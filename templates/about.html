03 // AUTOMATION FIRST

Built for automation.

Everything in the GUI is exposed via a streamlined POST API. Integrate scans into webhook chains, CI/CD runners, or custom SOAR playbooks. Returns standardized JSON. No authentication required on local instances.

:: ENDPOINTS ::

POST /api/scan SINGLE TARGET 
curl -X POST http://localhost:5000/api/scan \
  -H 'Content-Type: application/json' \
  -d '{
    "host": "example.com",
    "port": 443,
    "options": {
      "chain":     true,
      "protocols": true,
      "ciphers":   false,
      "headers":   true,
      "ct":        false,
      "dns":       true,
      "ports":     false
    }
  }'
POST /api/scan/bulk UP TO 25 HOSTS 
curl -X POST http://localhost:5000/api/scan/bulk \
  -H 'Content-Type: application/json' \
  -d '{
    "hosts": ["example.com", "github.com", "cloudflare.com"],
    "options": { "headers": true, "chain": true }
  }'
GET /report/<id>.json DOWNLOAD 
curl http://localhost:5000/report/abc123.json \
  > scan-report.json
GET /healthz LIVENESS 
curl http://localhost:5000/healthz
# {"status":"ok","time":"2026-05-24T..."}

04 // SELF-HOST

Runs in your network. No telemetry.

Your target endpoints stay yours. TLS.INSPECTOR ships zero analytics, stores no scan history beyond the in-memory deque, and operates offline inside firewalled subnets.

:: DOCKER ::

docker-compose.yml RECOMMENDED 
services:
  tls-inspector:
    build: .
    ports:
      - "5000:5000"
    restart: unless-stopped
DEPLOY COMMAND ONE-LINER 
docker compose up -d
